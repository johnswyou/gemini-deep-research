## Codebase Patterns

Read this section BEFORE editing. Rules discovered the hard way — don't
re-learn them.

### Tooling / lint
- **Ruff bugbear B008:** Typer's `typer.Option(...)` / `typer.Argument(...)`
  in default values trip B008. Fixed via
  `[tool.ruff.lint.flake8-bugbear] extend-immutable-calls = ["typer.Option", "typer.Argument"]`
  in `pyproject.toml`. DO NOT add `# noqa: B008` — use this config.
- **Ruff PLC0415 (import not at top):** only two exceptions are allowed
  project-wide. (a) `from google import genai` is lazy-imported inside
  `GdrClient.__init__` so `gdr --help` / `--version` stay fast. Use
  `# noqa: PLC0415` with a comment. (b) `src/gdr/cli.py` has a per-file
  ignore for PLC0415 (declared in `pyproject.toml`).
- **Ruff RUF001/RUF003 (ambiguous unicode):** en-dash `–` trips in strings
  and comments. Use ASCII hyphen `-`. Em-dash in docstrings is fine per
  existing code (they're not flagged).
- **Ruff RUF100 (unused noqa):** `A002` and `BLE001` are NOT in the lint
  set — don't add noqa for them.
- **Ruff PLR0915 (too many statements):** the 50-statement limit catches
  `run()` easily. Extract helpers like `_finalize_and_render`,
  `_consume_create_result`, `_safe_build_client`.
- **Ruff PLW2901 (for-loop var reassignment):** rename `stripped = line.strip()`
  to avoid reassigning loop var `line`.
- **Ruff RUF059 (unused unpack):** prefix unused tuple members with `_`:
  `agg, _events = _collect(...)`.
- **Ruff E402 (imports not at top):** don't put module-level assignments
  (like `_UTC = timezone.utc`) between imports. Put them after ALL imports.

### mypy (strict, target py310)
- `datetime.UTC` is 3.11+. Use `from datetime import timezone` and
  `timezone.utc` (aliased as `_UTC = timezone.utc` at module scope, after
  imports). This is project-wide.
- `tomli` import has a mypy override (only imported on 3.10 branch) in
  pyproject.
- google.genai has a mypy override (ignore_missing_imports).
- Function naming collision: `Store.list` shadows the builtin `list` in
  class scope, breaking `list[Record]` annotations. This class uses
  `Store.recent(...)` instead. DO NOT rename back.
- Cast `policy.redact(raw)` to `dict[str, Any]` — the redactor returns
  `Any` generically.

### Domain architecture
- **`execute_research()` is the shared execution pipeline.** Both
  `gdr research` (with/without `--plan`) and `gdr plan approve` call it.
  Do NOT duplicate create/poll/render logic. The function accepts
  `previous_interaction_id` and `api_input` for plan-approval cases.
  As of Phase 6 it also accepts derived-domain kwargs
  (`builtin_tools`, `mcp_servers`, `file_search`, `input_parts`,
  `visualization`, `untrusted_input`). The CLI layer parses flags via
  `core/inputs.py` into these types — keep parsing out of
  `execute_research`.
- **`core/inputs.py` is where CLI flags become domain objects.** Every
  parser raises `ConfigError` (never bare `ValueError`), so the CLI's
  try/except → exit code 4 is uniform. `commands/research.py` stays lean
  by delegating all flag parsing here.
- **Auto-untrusted mode fires from `--file` or `--url`** when
  `config.safe_untrusted=True` (the default). `--untrusted-input` also
  trips it manually. The resulting `SecurityPolicy(untrusted=True)` is
  the single place tools get filtered (`code_execution`, `mcp_server`).
  `file_search` is considered safe and stays.
- **Tool ordering in the wire shape is contractual:** builtin simple
  tools → `file_search` → `mcp_server`. Tests assert this explicitly
  (`TestCombinedToolsAndMultimodal`). Reordering changes how dry-runs
  and fixtures look downstream.
- **--mcp-header splits on FIRST `:`** so values like `Bearer abc:123`
  survive. `str.partition("=")` then `str.partition(":")` on the rest.
- **--file-search-store accepts bare names**; the CLI prepends
  `fileSearchStores/`. `FileSearchSpec` still enforces the prefix, so
  config-TOML users get the same guarantee.
- **Auto-append `url_context` tool** when `--url` is used *and* the user
  overrode `--tool`. Handled in `_parse_flag_inputs` via
  `ensure_url_context_tool`. Config defaults already include
  `url_context`, so the branch is only hit when the user narrows tools.
- **`commands/_common.py` is for Phase-7+ family helpers** (client
  build, record lookup, `parse_since`). Do NOT dump research-command
  helpers here — those stay in `commands/research.py`. Rule of thumb:
  if ≥2 command modules need it, factor it to `_common`; otherwise
  keep it local.
- **`parse_since` accepts three forms:** relative (`7d`, `24h`, `30m`,
  `2w`), date-only (`YYYY-MM-DD`), and full ISO 8601 (with `Z` or
  offset). Anything else raises `ConfigError`. The helper takes a `now`
  kwarg for deterministic tests.
- **`gdr resume` never overwrites artifacts by default.** On collision
  it writes to a sibling directory suffixed `_resumed_<ts>`. The user
  opts in to overwriting via `--force`. Record lookup is required — no
  local record means we can't reconstruct a `RunContext`, so the
  command exits 4 and suggests `gdr research`.
- **`gdr cancel` detects missing SDK method gracefully.** Older
  `google-genai` builds may not expose `interactions.cancel`. We check
  via `getattr(client.interactions, "cancel", None)` and print a clear
  "upgrade your SDK" message on absence (exit 4) rather than a raw
  AttributeError.
- **`gdr show` supports id prefix match.** If an exact id lookup misses,
  we try `store.recent()` filtered by prefix. Returns None on ambiguity
  so the user gets the "no record" error (avoids accidental wrong-run
  reads when prefixes collide).
- **Attribute-then-key `_get` helper** appears in three modules
  (`rendering.py`, `streaming.py`, `planning.py`). It bridges SDK typed
  objects (attribute access) and dict-shaped test mocks (key access) so
  tests can pass `SimpleNamespace` OR `dict` without a separate adapter.
- **Stream contract:** `interaction.complete` during streaming has
  `outputs=None` per Google's docs. The research command ALWAYS calls
  `.get(id=...)` after the stream ends — whether cleanly or by
  disconnect — to get authoritative outputs. Partial buffers are
  discarded on disconnect. Never reconstruct reports from the stream.
- **MCP header validation is eager.** `build_tools()` validates headers
  BEFORE any wire-shape assembly. One bad header aborts the whole
  request so partial kwargs never reach the API.
- **Untrusted-mode tool filtering runs LAST** in `build_tools()` so the
  `stripped` return list reflects what would actually have been sent.
- **Collaborative planning kwargs are minimal.** `build_plan_kwargs()`
  sets `thinking_summaries="none"` and `visualization="off"` — plans
  should return fast; streaming UI adds no value.

### Rendering
- When printing streamed text, use `console.print(text, end="",
  highlight=False, markup=False, soft_wrap=True)`. Model output may
  contain `[bold]` brackets that would otherwise crash Rich's markup
  parser.
- Text delta buffer in `LiveRenderer` flushes on newline. Use
  `finish()` at end of stream to empty the tail buffer.
- Use Rich's `Status` (not `Live`) for polling spinners — `Live` owns
  the viewport and wipes streamed prints above it.
- **Image outputs are extracted defensively.** `extract_images()` skips
  entries with missing `data` or unparseable base64 so garbage on the
  wire never aborts the full render. The raw transcript still captures
  the original payload for debugging.
- **Image filenames are `image_NNN<.ext>`** (3-digit zero-padded),
  extension from `mimetypes.guess_extension(mime)` with `.png` fallback.
  stdlib may pick `.jpe` for `image/jpeg`; tests check only the prefix.
- **Images section goes *between* body and Sources** in `report.md` —
  tests assert "## Images" before "## Sources" ordering.

### Storage + paths
- Store: `$GDR_STATE_DIR` → `$XDG_STATE_HOME/gdr/` → `~/.local/state/gdr/`.
- Config: `$GDR_CONFIG_PATH` → `$XDG_CONFIG_HOME/gdr/config.toml` →
  `~/.config/gdr/config.toml`.
- Intentionally NOT using `platformdirs` — predictability over platform
  idiom for terminal tooling.
- `env:VAR` in TOML is **prefix-only** (`"env:TOKEN"` works;
  `"Bearer env:TOKEN"` does not). Tested + documented contract.
- Slug sanitization: `[^a-z0-9]+ → -`, max 64 chars, empty falls back to
  `"query"`. id6 fragment (first 6 alnum of interaction id) appended.
  Dashes in id strip out — test IDs in fixtures use dash-free strings
  (`"intabcxyz123"`) so `intabc` is predictable.

### Testing
- Fake the SDK at `google.genai.Client` via `mocker.patch`. Tests pass
  `SimpleNamespace` or `MagicMock` objects; the `_get` helpers accept
  both shapes. See `tests/unit/test_research_command.py::_install_fake_sdk`.
- Golden-file fixtures for streaming live in
  `tests/fixtures/streams/*.jsonl` (one JSON event per line). These
  cover happy path / mid-stream error / out-of-order / disconnect /
  interleaved content.
- Mock `typer.prompt` via `mocker.patch("gdr.core.planning.typer.prompt",
  side_effect=[...])` for interactive-loop tests.
- Rich output capture: `Console(file=io.StringIO(), record=True,
  force_terminal=False, width=120)` then `console.export_text()`.
- All command tests use an autouse `_isolated_env` fixture setting
  `GDR_STATE_DIR` and `GDR_CONFIG_PATH` to tmp_path so tests never
  touch real user state.

## Phase 8 (NEXT)

**Scope per plan:**

> Operator commands — `config`, `doctor --fix`. Commit:
> `feat: config and doctor commands`.

### Implementation checklist

1. **`gdr config get|set|edit|path`** — TOML config management.
   - `gdr config path` → print `~/.config/gdr/config.toml` (respects
     `$GDR_CONFIG_PATH` and `$XDG_CONFIG_HOME`).
   - `gdr config get [KEY]` → print the full resolved config (YAML or
     pretty-printed TOML) or the value at `KEY` (dot-separated path,
     e.g. `mcp_servers.factset.url`).
   - `gdr config set KEY VALUE` → write `KEY = VALUE` into the config
     TOML, creating the file if missing. Preserve comments/whitespace
     via `tomlkit` if we want to be nice; `tomllib` is read-only.
   - `gdr config edit` → open `$EDITOR` (fallback: `vi`) on the config
     path, creating a minimal template if the file doesn't exist yet.

2. **`gdr doctor [--fix]`** — env/SDK/API-key/network/output-dir
   validation. Prints a list of checks with pass/fail/warning status.
   - Check: Python ≥ 3.10.
   - Check: `google-genai` installed and ≥ `MIN_GENAI_VERSION`.
   - Check: `GEMINI_API_KEY` available (env or config); if set, show a
     fingerprint (first 4 + last 4 chars).
   - Check: network reachability to `generativelanguage.googleapis.com`
     (1 DNS lookup, don't actually call the API).
   - Check: `output_dir` exists and is writable.
   - Check: `state_dir` exists and is writable.
   - With `--fix`: create a minimal `config.toml` if missing, create
     the output + state directories, and print next-step hints.

3. **New modules**
   - `commands/config.py` — Typer sub-app with 4 subcommands.
   - `commands/doctor.py` — a single Typer entry printing a table of
     check results.
   - Consider adding `tomlkit>=0.13` to dependencies for
     comment-preserving writes; fallback to plain string concatenation
     if we want to keep dependencies minimal.

4. **Tests** — roughly +20-25
   - `test_config_command.py` — get/set/path/edit (mock `$EDITOR`).
   - `test_doctor_command.py` — each check in a parametrized matrix,
     plus `--fix` happy path.

### Gotchas to watch for in Phase 8
- Writing TOML with comments is annoying. `tomllib` is read-only in
  stdlib; `tomlkit` preserves comments but adds a dep. For MVP we can
  use a naive string-formatter that regenerates the file from scratch
  — users lose their comments, but `gdr config set` is rarely used
  compared to hand-editing. Document the tradeoff.
- `gdr doctor` must NOT call the real API (would consume quota on a
  health check). Use DNS lookup or a lightweight HEAD to a public
  Google endpoint to probe network.
- `--fix` must be idempotent. Running `gdr doctor --fix` twice should
  be safe and produce the same state. Test this explicitly.
- `gdr config edit` with a missing `$EDITOR` should print a clear
  hint, not crash. Also sanitize the editor string (no shell
  injection) — use `subprocess.run([editor, path])` with a list.
- `gdr config set KEY VALUE` needs type inference: bool, int, string
  should map correctly. Quote strings with spaces or special chars.
  Reject keys that pydantic wouldn't accept by round-tripping through
  `Config.model_validate` after the write.

### What phases 9+ need from Phase 8
- Nothing structural. Phase 9 is docs (README/USAGE/MCP/examples);
  Phase 10 is CI + PyPI release. Neither touches runtime behavior.

    the underlying function lives in `follow_up.py` (Python naming).
  - `Enum` subclassing `str, Enum` works seamlessly with Typer as a
    `--part` choice; values are auto-documented in `--help`.
  - Tests that need a pre-seeded `JsonlStore` should write via the
    `JsonlStore.open(path)` API (not hand-craft the file) to avoid
    subtle format drift — the `Record.model_dump_json` shape is
    authoritative.
  - When typing Store protocol methods accept `protocol-only` calls,
    use the real `Store` import instead of `object` — mypy catches the
    difference and picks up return-type narrowing.
  - `mimetypes.guess_extension("image/jpeg")` returns `.jpe` on stdlib;
    tests covering filenames should match on prefix, not exact ext.
  - `ruff format` often reformats line-wrapping decisions on new files
    — run format AFTER writing, then the lint/mypy/test gate, to avoid
    noise in the final pass.
